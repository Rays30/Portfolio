OVERVIEW

Project Overview

Elemental Nexus is a co-operative browser platformer inspired by the classic Fireboy & Watergirl game — built entirely from scratch during a 4-hour hackathon using only HTML, CSS, and vanilla JavaScript. No frameworks, no game engines, no libraries. Just raw browser APIs and a ticking clock.

Two elemental characters — one tied to fire, one to water — must navigate a shared level, each immune to their own element but vulnerable to the other's hazards. Players must cooperate (or take turns controlling both characters) to solve environmental puzzles, avoid elemental traps, and reach the exit together. The game includes custom sprite animation, collision detection, tile-based level design, and a hand-crafted infographic screen explaining the game mechanics.

KEY FEATURES

  • Two-character elemental system — Fire and Water players with element-specific hazard immunity
  • Tile-based level design with environmental traps (fire pools, water pools, neutral hazards)
  • Custom canvas-based collision detection and gravity physics — built without any game engine
  • Dual-control keyboard scheme — both characters controllable simultaneously on one keyboard
  • Animated character sprites and environmental tiles
  • Infographic companion page explaining game mechanics and controls
  • Win and lose conditions with restart flow
  • Completed end-to-end in 4 hours under hackathon pressure

PROBLEM → SOLUTION

Problem: Build a playable, complete browser game in 4 hours with no external libraries or game engines — using only what the browser natively provides.

Solution: Stripped the design down to essential fun: two characters, one screen, elemental hazards, and a shared goal. Used HTML5 Canvas for rendering, wrote a minimal physics loop from scratch, and split time aggressively between gameplay feel and visual polish to ship something genuinely playable by the end of the hackathon.

CHALLENGES

  • Building a working collision system from scratch with no engine — handling tile-by-tile AABB detection under time pressure
  • Scoping the game aggressively — deciding what to cut so that what remained was actually fun and complete
  • Dual-keyboard controls that don't conflict — mapping two independent control schemes to one keyboard without input collisions
  • Canvas animation loop stability — keeping the game running at a consistent frame rate without requestAnimationFrame drift
  • Time management — balancing game logic, level design, UI polish, and the infographic page within 4 hours

LEARNINGS

  • How to build a game loop from scratch — requestAnimationFrame, delta time, and frame-rate independent movement
  • Tile-based collision detection fundamentals without relying on a physics library
  • Hackathon discipline — ruthless scoping, shipping early, and iterating fast under a hard deadline
  • How much you can accomplish with just HTML, CSS, and JavaScript when you focus on core mechanics
  • The importance of a fun core loop — get the feel right first, then add polish
