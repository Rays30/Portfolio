OVERVIEW

Project Overview

Minesweeper React + Firebase is a modern web reimagining of the classic Minesweeper game — rebuilt from scratch using React and backed by Firebase for real-time leaderboard persistence and user authentication. The project goes beyond a straight clone: it introduces custom gameplay mechanics, a unique visual flavor, and backend features that make the experience feel fresh and replayable.

Players can create accounts, compete on a global leaderboard ranked by fastest completion times, and experience signature twists like a guaranteed safe first click, a second-chance life system, and dynamic difficulty scaling tied to player performance. The app is fully responsive, playable on both desktop and mobile browsers.

KEY FEATURES

  • Classic Minesweeper gameplay — reveal safe tiles, flag mines, win or lose
  • Safe First Click — the first tile clicked is guaranteed mine-free
  • Second Chance Mode — players get one extra life before the game ends
  • Dynamic Difficulty — mine count scales with player performance over time
  • Firebase Leaderboard — top scores saved to Firestore and displayed in real time
  • User Accounts — sign in with Firebase Auth to track personal records
  • Color-coded number tiles for improved readability and visual clarity
  • Toast notifications for win, lose, and personal best events
  • Sound effects for tile clicks, flagging, and explosions (toggleable)
  • Animated cell reveals and smooth game state transitions
  • Fully responsive — playable on desktop and mobile browsers

PROBLEM → SOLUTION

Problem: Classic Minesweeper has no backend — scores disappear on refresh, there's no competition layer, and the infamous unfair first-click mine can end a game before it even starts.

Solution: Rebuilt the game with React for a reactive component-driven board, added Firebase Auth and Firestore to persist scores and power a live leaderboard, and introduced custom mechanics — safe first click, second chance lives, and dynamic difficulty — to make each game fairer and more engaging.

CHALLENGES

  • Implementing safe first-click — mines must be placed after the first click resolves, requiring a two-phase board generation approach
  • Designing dynamic difficulty that feels fair, not punishing — calibrating how aggressively mine count scales with win streaks
  • Real-time leaderboard with Firestore — handling concurrent writes and ordering scores correctly without duplicates
  • Making the game fully playable on mobile with touch events on a grid layout originally designed for mouse interaction

LEARNINGS

  • Deep React state management — modeling a 2D game board entirely in useState with efficient re-renders
  • Firebase Firestore real-time listeners and security rules for leaderboard integrity
  • Game design fundamentals — balancing fairness, tension, and replayability in a probability-based game
  • How auth and persistent data changes the entire user experience of even a simple game
  • Responsive game UI — adapting grid-based interfaces across screen sizes without breaking game logic
