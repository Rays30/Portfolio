OVERVIEW

Project Overview

Lichess Performance Analytics is a browser-based dashboard that turns raw chess game data into actionable performance insights — with no server, no signup, and no dependencies. I built it because I wanted to understand my patterns as a chess player: not just my win rate, but why I win and lose. Which openings actually work for me? Am I worse with Black? Do I lose more on timeout in Bullet? Most platforms give you a number — I wanted the analysis behind it.

The app ingests PGN exports from Lichess, parses and deduplicates games via hashing, extracts structured fields from unstructured text, and runs behavioral analysis across 9 dashboard pages. An insight engine then generates human-readable coaching statements — like "You resign too early" or "Your win rate with the Sicilian is 20% below your average" — complete with confidence ratings based on sample size.

KEY FEATURES

  • Upload PGN files, paste raw PGN text, or fetch games live from Lichess via their public API
  • 9 dashboard pages: color split, time control, openings, outcomes, game length, game phase, streaks, trends, and insights
  • PGN parser handles malformed records and deduplicates games via hashing
  • Cross-category correlation engine — e.g. "You lose significantly more on time in Bullet when playing Black"
  • Human-readable coaching statements generated by the insight engine
  • Confidence thresholds (High / Medium / Low) based on sample size — small samples are flagged, not hidden
  • All data stored in localStorage — nothing ever leaves your browser
  • Export your parsed game data as JSON anytime
  • No framework, no build step, no backend — runs anywhere

PROBLEM → SOLUTION

Problem: Chess platforms like Lichess show aggregate stats but don't explain the patterns behind them. A player can see their win rate but not why they're losing — whether it's a specific opening, time pressure, playing with Black, or a recurring streak pattern.

Solution: Built a full client-side data pipeline that parses raw PGN exports, extracts structured performance fields, and runs multi-dimensional analysis — surfacing specific, actionable insights rather than just numbers.

CHALLENGES

  • PGN files from different platforms use inconsistent tag names, date formats, and termination strings — normalizing messy real-world input was half the engineering work
  • Designing meaningful cross-category correlations without a query language or database — all in plain JavaScript
  • Preventing misleading insights from small sample sizes without hiding data entirely
  • Architecting a multi-module app (parser, extractor, analyzer, insight engine, UI) with clean separation and zero circular dependencies — all without a framework

LEARNINGS

  • Data quality is the real challenge — half the engineering effort went into parsing and normalizing inconsistent PGN input, which mirrors real-world data work more than any textbook exercise
  • Sample size matters analytically — adding confidence thresholds was a small code change but a major improvement in the quality of insights produced
  • Constraints force better architecture — building without a framework meant thinking carefully about module boundaries, data flow, and state management from scratch
  • Client-side data pipelines are viable — the entire parse → extract → analyze → visualize flow runs in the browser with no performance issues
  • Designing for exploratory data analysis — users don't always know what they're looking for, so the UI had to make discovery feel natural across 9 different views
